## 上传到 GitHub

### 1. 创建 GitHub 仓库

访问：https://github.com/new

- Repository name: `dub-mobile-app`
- 不要勾选 "Initialize with README"

### 2. 推送代码

```bash
cd /d/project/互动视频/dub-mobile-app

# 添加远程仓库（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/dub-mobile-app.git

# 推送
git branch -M main
git push -u origin main
```

### 3. 启用 GitHub Actions

- 访问仓库的 Actions 标签
- 点击启用 workflows
- 自动开始构建 APK

### 4. 下载 APK

- 构建完成后（约 5-10 分钟）
- Actions → 最新的 workflow → Artifacts → 下载 `app-release`

## Deep Link 使用

### URL 格式

```
dubeditor://open?url=<目标网址>
```

### Web 页面调用示例

```javascript
const targetUrl = 'https://example.com/editor';
const appUrl = `dubeditor://open?url=${encodeURIComponent(targetUrl)}`;
window.location.href = appUrl;
```

## 与 uni-app 对比总结

**Capacitor（当前方案）优势：**
- ✅ 无需重写现有代码
- ✅ 直接使用 Web 应用
- ✅ 配置简单直接
- ✅ 完全控制 WebView
- ✅ 适合纯壳子场景

**适用场景：**
你的需求是"从浏览器跳转到 App，用 App 加载网址"，这正是 Capacitor 的强项。不需要 uni-app 的复杂功能。
