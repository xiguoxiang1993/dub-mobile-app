# 使用说明

## 这是什么？

这是一个**纯壳子应用**，不包含任何编辑器代码。它的作用是：

1. 从 Web 页面接收编辑器 URL
2. 在无浏览器 UI 的全屏环境中加载编辑器
3. 解决 iPad Safari 的所有手势冲突问题

## 工作流程

```
Web 页面 → 点击"在 App 中打开" → Deep Link 传递 URL → App 加载编辑器
```

## Web 端集成（3 步）

### 1. 添加按钮

```html
<button onclick="openInApp()">在 App 中打开编辑器</button>
```

### 2. 添加 JavaScript

```javascript
function openInApp() {
  // 获取当前编辑器 URL（包含所有参数）
  const editorUrl = window.location.href;

  // 构建 Deep Link
  const deepLink = `dubeditor://open?url=${encodeURIComponent(editorUrl)}`;

  // 打开 App
  window.location.href = deepLink;

  // 2 秒后如果还在页面，说明没安装 App
  setTimeout(() => {
    if (confirm('未检测到 App，是否下载？')) {
      window.location.href = '/download/dub-editor.apk';
    }
  }, 2000);
}
```

### 3. 完成

用户点击按钮后：
- 已安装 App → 自动打开 App 并加载编辑器
- 未安装 App → 提示下载

## 测试

### 在电脑上测试

1. 打开 `web-example.html`
2. 输入任意编辑器地址
3. 点击按钮查看生成的 Deep Link

### 在 Android 设备上测试

1. 安装 APK
2. 使用 ADB 命令测试：

```bash
adb shell am start -W -a android.intent.action.VIEW -d "dubeditor://open?url=https://example.com/editor"
```

3. 或在浏览器中访问你的 Web 页面，点击"在 App 中打开"

## URL 格式

```
dubeditor://open?url=<编辑器完整URL>
```

示例：
```
dubeditor://open?url=https://your-domain.com/editor?id=123&token=abc
```

**注意**：URL 必须经过 `encodeURIComponent()` 编码。

## 常见问题

### Q: 如何修改 URL Scheme？

编辑 `android/app/src/main/AndroidManifest.xml`，找到：

```xml
<data android:scheme="dubeditor" />
```

改为你想要的名称，例如：

```xml
<data android:scheme="myapp" />
```

然后重新构建 APK。

### Q: 可以传递多个参数吗？

可以，但建议把所有参数都放在编辑器 URL 中：

```javascript
const editorUrl = 'https://your-domain.com/editor?id=123&token=abc&mode=edit';
const deepLink = `dubeditor://open?url=${encodeURIComponent(editorUrl)}`;
```

### Q: 如何自定义 App 图标？

查看 [ASSETS.md](./ASSETS.md) 文档。

### Q: 支持 iOS 吗？

支持，但需要 Mac 电脑来构建。步骤：

```bash
npm run add:ios
npm run ios
```

在 Xcode 中配置签名并构建。
